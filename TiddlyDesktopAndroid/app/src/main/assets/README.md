# assets — bundled TiddlyWiki payload (mostly not checked in)

Checked in:
- `bridge/collab-bridge.js` — the collab plugin `window._nwjs*` shim (see CollabBridge.kt).

Generated by the Gradle asset-packaging tasks (git-ignored — see `app/build.gradle.kts`):

- `tiddlywiki.zip` — the TiddlyWiki5 engine, root entries `tiddlywiki/…`. Extracted on first
  run to `{filesDir}/tiddlywiki/` by `node/NodeEnvironment.kt` (`ensureResourcesExtracted`).
  Zipped by `packageTiddlyWikiAssets` from `../source/tiddlywiki` (override with
  `-PtdEngineDir=/abs/path`). Must contain `tiddlywiki.js` at the `tiddlywiki/` root.

- `wikilist.zip` — the WikiList landing UI as a TiddlyWiki **folder wiki** (root entries
  `wikilist/…`). Extracted to `{filesDir}/wikilist/` and served by a Node `--listen` server
  (`MainActivity`). Zipped by `packageWikiListAsset` from `packaging/wikilist/`, which is
  assembled from the classic `plugins/tiddlydesktop` + the Android `$tw.desktop` bridge by:

  ```sh
  sh packaging/build-wikilist.sh
  ```

Both tasks run automatically before the build (`preBuild` depends on them). Keep the node
binary OUT of these zips — it must live in `jniLibs/` to be executable.
